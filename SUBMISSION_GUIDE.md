# 发给老师的建议

## 1. 最推荐的交付方式

建议你把项目整理成一个压缩包发给老师，内容包括：

- 完整项目代码
- 使用教程
- 必要的运行截图
- 演示说明

建议压缩包命名：

```text
PDE_Solver_Project_Thomas.zip
```

## 2. 压缩包里建议保留的内容

建议保留：

- 源代码目录
- `requirements.txt`
- `QUICKSTART.md`
- `docs/API_GUIDE.md`
- `static/dist`
- `config`

建议删除后再打包：

- `.git`
- `.pytest_cache`
- `__pycache__`
- `log`
- `result`
- 临时测试脚本

如果你不确定哪些测试脚本可以删，最稳妥做法是先完整保留，再额外给老师一份“主要入口说明”。

## 3. 发老师时建议附上的说明

你可以直接发这样一段：

```text
老师您好，这是我的 PDE 求解器课程项目。

项目包含：
1. PDE 参数化求解
2. 特征提取与算法选择
3. 基于大模型的题目解析与自动求解
4. Web 前端演示界面与 Swagger API 文档

运行方法见压缩包内 QUICKSTART.md。

推荐演示入口：
- Web 页面：http://127.0.0.1:8001/app/
- API 文档：http://127.0.0.1:8001/docs

推荐测试题：
“求解一维热传导方程 u_t = k * u_xx，在区间 [0,1] 上，边界条件 u(0,t)=u(1,t)=0，初始条件 u(x,0)=sin(pi x)”

说明：当前系统面向 PDE 题目，普通力学题不属于自动求解支持范围。
```

## 4. 老师如果不想自己装环境

你可以额外提供：

- 一段 2 到 5 分钟的演示录屏
- 3 到 5 张关键截图

建议截图包括：

- 首页
- LLM 配置页
- 题目解析成功页
- 自动求解成功页
- Swagger 文档页

## 5. 最稳妥的展示方式

如果老师是现场看，建议你自己电脑演示。  
如果老师是课后查看，建议：

1. 发压缩包
2. 发 `QUICKSTART.md`
3. 再附一个演示视频

这样老师即使不亲自运行，也能快速看懂成果。

## 6. 你现在可以怎么做

建议顺序：

1. 确认项目能正常启动
2. 用一条 PDE 样例题跑通“解析题目”和“自动求解”
3. 截图
4. 打包项目
5. 把 `QUICKSTART.md` 和一段说明一起发给老师
