# 项目差异与后续优化总纲

## 1. 与最初路线图相比，当前项目已经做到的部分

### 1.1 基础层

已完成：

- 一维热传导方程 `heat1d`
- 一维波动方程 `wave1d`
- 二维非线性 Poisson 方程 `poisson2d_nonlinear`
- FastAPI 后端服务
- React 前端演示页
- WebSocket 实时求解进度
- 特征提取、算法选择、结果评估完整闭环

与最初路线图的差异：

- 你最初路线图里强调“从纯 Python + NumPy/SciPy 基础数值框架起步”，这一点现在已经实现
- 但当前仍以“可运行课程项目框架”为主，还不是一个大规模科研级求解平台

### 1.2 AI 模块

已完成：

- `static_rf`：随机森林算法选择器
- `static_xgb`：XGBoost 算法选择器
- `dynamic_rl`：强化学习策略选择器
- `mlp_nn`：轻量 MLP 神经网络算法选择增强模块

与最初路线图的差异：

- 最初路线图里的“AI 模块”更偏完整研究路线，包括 GNN、PINN、自演化反馈
- 当前实际落地的是“算法选择增强”方向，而不是“神经网络直接求 PDE 解”
- 这是更务实也更适合课程汇报的版本

### 1.3 后端与接口

已完成：

- `/extract_feature`
- `/select_algorithm`
- `/solve_equation`
- `/evaluate_result`
- `/api/parse_question`
- `/api/auto_solve`
- `/ws/solve/{task_id}`

与最初路线图的差异：

- 接口闭环已经搭起来
- 但模型评估和自动优化还偏“离线代理型”，不是在线自治系统

### 1.4 前端展示

已完成：

- 参数化求解页
- 题目解析页
- LLM 配置页
- 可视化结果展示
- WebSocket 进度条
- “算法策略 + 智能推荐算法”交互

与最初路线图的差异：

- 你最初图里偏 Streamlit 风格快速验证
- 当前实际走的是 React + FastAPI 一体化展示
- 展示效果和课程项目完整度反而更强

## 2. 当前项目的主要短板

### 2.1 数值求解层

- `wave1d` 当前稳定实现的是 FDM
- `wave1d` 还没有真正接入 FEM / Spectral 的数值求解版本
- `poisson2d_nonlinear` 还是演示型 manufactured-solution baseline

### 2.2 算法选择层

- `RF` 和 `MLP` 当前在代表性合成数据集上效果很好
- `dynamic_rl` 仍然偏演示版，当前准确率明显低于监督学习策略
- 训练数据仍主要来自 synthetic dataset，不是真实工程数据

### 2.3 题目解析层

- 大模型不可用时会自动降级到规则解析
- 规则解析可演示，但题意理解仍比较粗糙
- 非 PDE 题虽然已有拦截，但复杂题仍可能误判

### 2.4 工程化层

- 存在较多零散测试脚本
- 警告和部分历史文件还未完全收干净
- 基准报告虽然已有，但还没完全接入文档与展示流程

## 3. 后续优化总纲

### 第一阶段：稳定当前版本

目标：

- 把现有功能打磨到“演示稳定、结果一致、文档清晰”

重点：

- 补充 `wave1d` 的结果解释与界面说明
- 继续清理零散测试脚本
- 整理 benchmark 结果写入文档
- 统一 README / QUICKSTART / SUBMISSION 文档口径

### 第二阶段：补齐算法与方程能力

目标：

- 让“方程类型”与“算法选择”关系更完整

重点：

- 为 `wave1d` 增加更多求解器实现
- 给 `poisson2d_nonlinear` 增加更强的非线性求解策略
- 让 `selected.algorithm_key` 和“实际执行算法”彻底一一对应

### 第三阶段：增强 AI 模块可信度

目标：

- 让 AI 模块不只是“会推荐”，而是“推荐结果更可解释、更可靠”

重点：

- 扩展 synthetic dataset
- 增加 benchmark 数据集
- 增加混淆矩阵、准确率、稳定性报告
- 继续优化 `dynamic_rl`

### 第四阶段：向研究型能力扩展

目标：

- 接近你最初路线图里的更强 AI PDE 方向

重点：

- 引入 GNN 做更复杂场景的算法选择
- 尝试 PINN / DeepONet 原型
- 加入在线反馈与持续训练机制

## 4. 推荐的实际执行顺序

1. 稳定现有 `heat1d / wave1d / poisson2d_nonlinear`
2. 固化 benchmark 和实验报告
3. 强化 `mlp_nn` 与 `dynamic_rl`
4. 再扩展到 GNN / PINN

## 5. 当前最值得优先优化的项目

- `dynamic_rl` 准确率偏低
- `wave1d` 算法实现与策略映射还可以继续增强
- benchmark 结果还没有完全变成老师能一眼看懂的实验结论
- 文档虽然已可用，但还可以进一步整理成“论文/汇报结构”
